<task type="auto">
  <name>Task 1: Rewrite alert system queries to ORM</name>
  <files>mesher/storage/queries.mpl</files>
  <action>
Rewrite these alert query functions from Repo.query_raw/execute_raw to ORM APIs. Plan 01 already rewrote check_new_issue and toggle_alert_rule if they were in its scope -- verify first and skip if already done.

Available ORM APIs: Query.from, Query.where, Query.where_raw (? placeholders), Query.select_raw, Query.order_by, Query.order_by_raw, Query.limit, Query.join_as, Repo.all, Repo.update_where, Repo.delete_where.

**Rewrite to ORM (up to 7 functions):**

1. `list_alert_rules` -- Query.from(AlertRule.__table__()) + Query.where_raw("project_id = ?::uuid", [project_id]) + Query.order_by(:created_at, :desc) + Query.select_raw(["id::text", "project_id::text", "name", "condition_json::text", "action_json::text", "enabled::text", "cooldown_minutes::text", "COALESCE(last_fired_at::text, '') AS last_fired_at", "created_at::text"]) + Repo.all.

2. `get_event_alert_rules` -- Query.from(AlertRule.__table__()) + Query.where_raw("project_id = ?::uuid AND enabled = true AND condition_json->>'condition_type' = ?", [project_id, condition_type]) + Query.select_raw(["id::text", "name", "cooldown_minutes::text"]) + Repo.all.

3. `should_fire_by_cooldown` -- Query.from(AlertRule.__table__()) + Query.where_raw("id = ?::uuid AND (last_fired_at IS NULL OR last_fired_at < now() - interval '1 minute' * ?::int)", [rule_id, cooldown_str]) + Query.select_raw(["1 AS ok"]) + Repo.all, then check List.length > 0.

4. `get_threshold_rules` -- Query.from(AlertRule.__table__()) + Query.where_raw("enabled = true AND condition_json->>'condition_type' = 'threshold'", []) + Query.select_raw(["id::text", "project_id::text", "name", "condition_json::text", "cooldown_minutes::text"]) + Repo.all.

5. `list_alerts` -- Query.from(Alert.__table__()) + Query.join_as(:inner, AlertRule.__table__(), "r", "r.id = alerts.rule_id") + Query.where_raw("alerts.project_id = ?::uuid AND (? = '' OR alerts.status = ?)", [project_id, status, status]) + Query.select_raw([...all columns with qualified table names...]) + Query.order_by_raw("alerts.triggered_at DESC") + Query.limit(50) + Repo.all.

Note: The optional status filter uses ($2 = '' OR alerts.status = $2) which passes status twice. With ORM where_raw using ? placeholders, pass [project_id, status, status] to bind both positions.

6. `toggle_alert_rule` (if not done in Plan 01) -- Repo.update_where with Query.where_raw("id = ?::uuid", [rule_id]) + %{"enabled" => enabled_str}. Return Ok(1) after success.

7. `check_new_issue` (if not done in Plan 01) -- Query.where_raw("id = ?::uuid AND first_seen = last_seen", [issue_id]) + select_raw(["1 AS is_new"]) + Repo.all, check List.length > 0.

**Document ORM boundaries (3 functions):**

1. `create_alert_rule` -- "ORM boundary: INSERT...SELECT with server-side JSONB extraction from parameter ($2::jsonb) and COALESCE defaults. Repo.insert takes Map<String,String> of literal values but this query extracts and transforms fields from a JSONB parameter server-side. Intentional raw SQL."

2. `evaluate_threshold_rule` -- "ORM boundary: Cross-join between two derived tables (event count subquery + cooldown subquery) with CASE expression, interval arithmetic, and multiple bound parameters in complex expressions. Not expressible via ORM query builder. Intentional raw SQL."

3. `fire_alert` -- "ORM boundary: INSERT with jsonb_build_object() for computed JSONB column plus follow-up UPDATE of last_fired_at = now(). Could split into two-step ORM pattern but jsonb_build_object in INSERT VALUES is not expressible via Repo.insert Map<String,String>. Intentional raw SQL."
  </action>
