  <action>
Rewrite these query functions from Repo.query_raw/execute_raw to ORM APIs. Follow established patterns from Phase 110-111 (Query.from + pipe chain + Repo.all/Repo.update_where). Query.limit takes Int -- parse limit_str with String.to_int/Option before passing.

Available ORM APIs: Query.from, Query.where, Query.where_raw (with ? placeholders), Query.select_raw (List of column expressions), Query.order_by(:field, :asc/:desc), Query.order_by_raw("expr"), Query.group_by(:field), Query.group_by_raw("expr"), Query.limit(Int), Query.offset(Int), Query.join_as(:inner/:left, table, alias, on_clause), Repo.all, Repo.update_where, Repo.delete_where.

**Rewrite to ORM (13 functions):**

1. `filter_events_by_tag` -- Query.where_raw for "project_id = ?::uuid", "tags @> ?::jsonb", "received_at > now() - interval '24 hours'" + select_raw + order_by + limit. Parse limit_str to Int for Query.limit.

2. `list_events_for_issue` -- Query.where_raw for "issue_id = ?::uuid" + conditional cursor where_raw for "(received_at, id) < (?::timestamptz, ?::uuid)" + select_raw + order_by_raw("received_at DESC, id DESC") + limit. Parse limit_str.

3. `event_volume_hourly` -- Query.where_raw for "project_id = ?::uuid" and "received_at > now() - interval '24 hours'" + select_raw with string-interpolated bucket: ["date_trunc('" <> bucket <> "', received_at)::text AS bucket", "count(*)::text AS count"] + group_by_raw("1") + order_by_raw("1"). Bucket is validated by caller (only "hour"/"day").

4. `error_breakdown_by_level` -- Query.where_raw for project_id + 24h interval + select_raw(["level", "count(*)::text AS count"]) + group_by_raw("level") + order_by_raw("count DESC").

5. `top_issues_by_frequency` -- Query.where_raw for "project_id = ?::uuid" + Query.where(:status, "unresolved") + order_by(:event_count, :desc) + select_raw + limit. Parse limit_str.

6. `issue_event_timeline` -- Query.where_raw for "issue_id = ?::uuid" + order_by(:received_at, :desc) + select_raw + limit. Parse limit_str.

7. `get_event_detail` -- Query.where_raw for "id = ?::uuid" + select_raw with all COALESCE expressions.

8. `list_api_keys` -- Query.where_raw for "project_id = ?::uuid" + order_by(:created_at, :desc) + select_raw with COALESCE for revoked_at.

9. `get_members_with_users` -- Query.join_as(:inner, User.__table__(), "u", "u.id = org_memberships.user_id") + Query.where_raw for "org_memberships.org_id = ?::uuid" + select_raw with qualified columns + order_by_raw("org_memberships.joined_at").

10. `update_member_role` -- Repo.update_where with Query.where_raw("id = ?::uuid AND ? IN ('owner', 'admin', 'member')", [membership_id, new_role]) and update map %{"role" => new_role}. Change return to use Repo.update_where result pattern (it returns Result<Map, String> per Phase 110 fix -- extract count or return Ok(1)). Actually, the original returns Int!String from execute_raw. With Repo.update_where the return is Result<Map, String>. Since the caller expects Int!String, adapt: call Repo.update_where, on Ok(_) return Ok(1), on Err(e) return Err(e). Actually examine the exact return: Repo.update_where returns Result<Map<String,String>, String>. Need to handle this.

11. `check_new_issue` -- Query.where_raw("id = ?::uuid AND first_seen = last_seen", [issue_id]) + select_raw(["1 AS is_new"]) + Repo.all, check List.length.

12. `acknowledge_alert` -- Repo.update_where with where_raw("id = ?::uuid AND status = 'active'", [alert_id]) and update map. But the SET includes "acknowledged_at = now()" -- now() is a PG server-side function. Repo.update_where takes Map<String,String> which can only set literal values. Cannot express now(). Two options:
    - Two-step pattern (like revoke_api_key): query now()::text, then update_where with the timestamp string.
    - Keep as execute_raw with ORM boundary doc.
    DECISION: Keep as execute_raw with ORM boundary doc. "ORM boundary: Repo.update_where Map<String,String> cannot express server-side now() function call in SET clause. Intentional raw SQL."

13. `resolve_fired_alert` -- Same now() issue as acknowledge_alert. Keep as execute_raw with ORM boundary doc.

14. `toggle_alert_rule` -- Repo.update_where with where_raw("id = ?::uuid", [rule_id]) and update map %{"enabled" => enabled_str}. Same return type adaptation as update_member_role.

Revised after analysis: 11 queries rewrite to ORM, 2 alert status transitions (acknowledge_alert, resolve_fired_alert) get boundary docs instead.

For each ORM boundary, add comment above the function doc. Format: "# ORM boundary: [explanation]. Intentional raw SQL."
