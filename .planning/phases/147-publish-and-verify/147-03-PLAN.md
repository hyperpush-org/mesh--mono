**Step 1: Create the consumer project directory**
```bash
mkdir -p tests/e2e/consumer
```

**Step 2: Create mesh.toml** for the consumer project:
```toml
[package]
name = "consumer"
version = "0.1.0"
description = "Test consumer for mesh-slug package"

[dependencies]
"{github_login}/mesh-slug" = "1.0.0"
```
(Replace `{github_login}` with the actual login from Plan 02 SUMMARY, e.g. `"snowdamiz/mesh-slug" = "1.0.0"`)

**Step 3: Create main.mpl** for the consumer program:
```
from Slug import slugify

fn main() do
  let result = slugify("Hello World!")
  println(result)
end
```

**Step 4: Install mesh-slug** using the built meshpkg binary:
```bash
./target/debug/meshpkg install --registry https://registry.meshlang.dev tests/e2e/consumer
```
Or equivalently (using default registry):
```bash
cd tests/e2e/consumer && ../../../target/debug/meshpkg install
```

Expected output:
```
✓ Installed {github_login}/mesh-slug@1.0.0
✓ Updated mesh.lock
```

After install, verify the package is on disk:
```bash
ls tests/e2e/consumer/.mesh/packages/
```
Must show a directory named `{github_login}/mesh-slug@1.0.0` (or similar — the exact format is `{name}@{version}`).

Verify slug.mpl is present:
```bash
ls tests/e2e/consumer/.mesh/packages/{github_login}/mesh-slug@1.0.0/
```
Must contain `slug.mpl`, `mesh.toml`, `main.mpl`.

**Step 5: Compile the consumer program**:
```bash
./target/debug/meshc build tests/e2e/consumer
```

Expected output:
```
  Compiled: tests/e2e/consumer/consumer
```

If compilation fails with "ImportModuleNotFound: Slug" — Plan 01's discovery.rs fix did not take effect. Check that `cargo build -p meshc` was re-run and the new binary is in `./target/debug/meshc`.

**Step 6: Run the compiled binary**:
```bash
tests/e2e/consumer/consumer
```

Expected output:
```
hello-world
```
(slugify("Hello World!") = "hello-world" — lowercase, spaces replaced with hyphens)
