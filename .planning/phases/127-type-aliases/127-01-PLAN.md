```rust
impl TypeAliasDef {
    /// The alias name.
    pub fn name(&self) -> Option<Name> {
        child_node(&self.syntax)
    }

    /// The visibility modifier (`pub`), if present.
    pub fn visibility(&self) -> Option<Visibility> {
        child_node(&self.syntax)
    }

    /// The target type annotation node in the AST (after `=`).
    /// Returns the first child SyntaxNode that is not NAME, VISIBILITY, or GENERIC_PARAM_LIST.
    pub fn target_type_name(&self) -> Option<String> {
        use crate::syntax_kind::SyntaxKind;
        // Collect IDENT tokens that appear after the EQ sign
        let mut past_eq = false;
        for child in self.syntax.children_with_tokens() {
            match child {
                rowan::NodeOrToken::Token(t) => {
                    if t.kind() == SyntaxKind::EQ {
                        past_eq = true;
                        continue;
                    }
                    if past_eq && t.kind() == SyntaxKind::IDENT {
                        return Some(t.text().to_string());
                    }
                }
                _ => {}
            }
        }
        None
    }
}
```

**1b. Fix `parse_type_alias` in `compiler/mesh-parser/src/parser/items.rs`** to call `parse_optional_visibility(p)` before consuming `TYPE_KW`. Currently `parse_type_alias` starts with `p.advance() // TYPE_KW` which leaves the `pub` token unconsumed when called from the `PUB_KW` dispatcher branch.

Change (around line 1046):
```rust
pub(crate) fn parse_type_alias(p: &mut Parser) {
    let m = p.open();

    // Optional visibility (handles `pub type Alias = T`).
    parse_optional_visibility(p);

    p.advance(); // TYPE_KW

    // Type name.
    if p.at(SyntaxKind::IDENT) {
        let name = p.open();
        p.advance();
        p.close(name, SyntaxKind::NAME);
    } else {
        p.error("expected type alias name");
        p.close(m, SyntaxKind::TYPE_ALIAS_DEF);
        return;
    }

    // Optional type parameters: <T>
    if p.at(SyntaxKind::LT) {
        parse_generic_param_list(p);
    }

    // Expect `=`.
    p.expect(SyntaxKind::EQ);

    // Parse the aliased type.
    if !p.has_error() {
        parse_type(p);
    }

    p.close(m, SyntaxKind::TYPE_ALIAS_DEF);
}
```

After this change, `pub type Url = String` parses to a `TYPE_ALIAS_DEF` node with a `VISIBILITY` child (containing `pub`) followed by `NAME` (`Url`) followed by `EQ` followed by `IDENT` (`String`).

**2a. Add undefined-alias-type validation in `compiler/mesh-typeck/src/infer.rs`.**

After ALL items in a source file have been pre-registered (struct defs, sum types, etc.), add a validation pass that checks each registered type alias's target type against known types. This must run after pre-registration so that `type A = B` where `B` is defined later in the file still resolves correctly.

In `register_type_alias` (lines 3131-3157), the function currently registers without validation. The aliased type is parsed by `parse_alias_type`. Add a post-registration validation helper:

```rust
/// Validate all registered type aliases have resolvable target types.
/// Called after all type registrations are complete.
fn validate_type_aliases(
    type_registry: &TypeRegistry,
    alias_defs: &[(TypeAliasDef, rowan::TextRange)],
    errors: &mut Vec<TypeError>,
) {
    for (alias_def, range) in alias_defs {
        let target_name = alias_def.target_type_name();
        if let Some(name) = target_name {
            // Check if target resolves to a known type:
            // builtin primitive, struct, sum type, or another alias
            let known = is_known_type(&name, type_registry);
            if !known {
                errors.push(TypeError::UndefinedType {
                    name,
                    range: *range,
                });
            }
        }
    }
}

fn is_known_type(name: &str, type_registry: &TypeRegistry) -> bool {
    // Primitives
    matches!(name, "Int" | "Float" | "String" | "Bool" | "Unit" |
                   "Option" | "Result" | "List" | "Map" | "Set" |
                   "Range" | "Queue" | "Pid" | "Atom" | "Regex" | "Never") ||
    type_registry.struct_defs.contains_key(name) ||
    type_registry.sum_type_defs.contains_key(name) ||
    type_registry.type_aliases.contains_key(name)
}
```

Wire this validation into the `infer` function after the pre-registration pass. You'll need to collect `(TypeAliasDef, TextRange)` tuples during the pre-registration loop and pass them to the validator.
