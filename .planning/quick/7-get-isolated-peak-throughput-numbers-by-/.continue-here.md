<context>
The hey binary at `hey-release.s3.us-east-2.amazonaws.com/hey_linux_amd64` does NOT output "X% in Y secs" percentile lines for working runs. Confirmed via `cat -A` debug: only "Latency distribution:$" header appears, zero percentile lines follow.

The `hey -o csv` flag outputs one row per request:
```
response-time,DNS+dialup,DNS,Request-write,Response-delay,Response-read,status-code,offset
0.0041,0.0000,0.0000,0.0000,0.0040,0.0001,200,0.0000
```
Column 1 = total response time (secs), column 7 = status code.

The fix requires changing `run_hey` to use `-o csv` and computing stats from the CSV:
- `rps` = count of all rows (NR-1 for header) / BENCH_DURATION
- `p50` = 50th percentile of column 1 (status=200 only) via sort
- `p99` = 99th percentile of column 1 (status=200 only) via sort

- Has `awk '/50% in/'` pattern — REPLACE WITH CSV approach
- Has all other fixes applied (--force, IPv6)

3. Replace the rps/p50/p99 computation block with CSV-based parsing:

```bash
csv=$(run_hey "$url" "$BENCH_DURATION")
rps=$(echo "$csv" | awk -F',' 'NR>1{n++} END{printf "%.4f", n/BENCH_DUR}' BENCH_DUR="$BENCH_DURATION")
p50_p99=$(echo "$csv" | awk -F',' 'NR>1 && $7==200{print $1}' | sort -n | awk \
  '{a[NR]=$1} END{n=NR; if(n>0) printf "%.4f secs|%.4f secs", a[int(n*0.50)+1], a[int(n*0.99)+1]; else printf "N/A|N/A"}')
p50=$(echo "$p50_p99" | cut -d'|' -f1)
p99=$(echo "$p50_p99" | cut -d'|' -f2)
```
